01 — Executive summary

What this system does

This project is a compact, education-oriented plagiarism detection web application. Users upload documents through a Flask-powered interface; the backend normalizes heterogeneous formats into plain text, then applies a classical fingerprinting pipeline (preprocessing → k-grams → hashing → winnowing → comparison) to surface overlap between corpora.

Problem domain

Academic or editorial scenarios where pairwise or small-batch document similarity is enough — not a large-scale search engine over billions of shards.

Design posture

Clear separation: ingestion utilities, matching logic, and HTTP orchestration live in different modules, which keeps the teaching surface area readable.

02 — System architecture

Layered view

The stack is intentionally thin: a single Flask entrypoint coordinates uploads and renders results, while utils/ hosts conversion and core matching routines.

04 — End-to-end data flow

Request lifecycle

From user gesture to rendered similarity signal.

1. Upload & route handling

The browser posts files to Flask routes in test_app.py; the app validates and stores transient upload state.

2. Canonical text extraction

file_converter.py maps PDF, DOCX, and TXT into a unified UTF-8 text representation suitable for tokenization.

3. Linguistic normalization

preprocess_text() reduces noise (e.g., punctuation stripping, case folding) so hashing is stable across stylistic variance.

4. k-gram construction

Sliding windows of length k capture local n-gram structure; this is the statistical substrate for overlap detection.

5. Hashing & winnowing

Hashes compress k-grams; winnowing selects a sparse, representative subset of fingerprints to keep comparisons tractable.

6. Fingerprint comparison

compare_fingerprints() measures multiset overlap (or related metric) between documents’ fingerprint streams.

7. Presentation

Results bubble back through Flask templates/JSON responses for human-readable reporting in the UI.

05 — Algorithm profile

Why this approach works

k-grams

Local shingles make the method resilient to small insertions compared to whole-file hashing, while still being simpler than dense neural embeddings.

Winnowing / fingerprints

Sublinear footprint: you compare representative sketches instead of every k-gram, which is essential for interactive turnaround on a laptop-class process.

Analyst note: For portfolio or coursework repos, consider renaming test_*.py files that are not automated tests — the test_ prefix often signals pytest/unittest discovery and can confuse CI and reviewers.

08 — Maturity recommendations

Elevating the project (optional roadmap)

  • Testing: Add pytest coverage for converters and fingerprint edge cases (empty files, unicode, scanned PDFs).
  • Security: Harden uploads (size caps, MIME sniffing, sandboxed extraction, virus scanning hook).
  • Observability: Structured logs + request IDs; timing spans around conversion vs. match phases.
  • Packaging: Dockerfile + gunicorn production profile; environment-based config.
  • UX: Progress states for long PDFs; downloadable JSON report of matched spans if available from the algorithm.
